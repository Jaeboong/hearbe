## 시나리오 2: 마이페이지 기능

### 2-1. 마이페이지 접속

- **URL**: `/B/member-info`

- **화면**: 회원 정보, 복지카드 관리, 주문내역, 위시리스트 접근 가능

### 2-2. 주문내역 확인

- **URL**: `/B/order-history`
- **동작**: 마이페이지에서 [주문내역] 클릭
- **결과**: 이전 주문 목록 표시

### 2-3. 위시리스트 확인

- **URL**: `/B/wishlist`
- **동작**: 마이페이지에서 [위시리스트] 클릭
- **결과**: 저장된 관심 상품 목록 표시

### 2-4. 복지카드 관리

- **URL**: `/B/card-management`
- **동작**: 마이페이지에서 [복지카드 관리] 클릭

## 시나리오 3: 회원가입 + 복지카드 OCR

### 3-1. 회원가입 페이지 진입

- **URL**: `/B/signup` (B형) 또는 `/C/signup` (C형)
- **동작**: 로그인 페이지에서 [회원가입] 클릭

### 3-2. 복지카드 OCR 촬영

- **화면**: 회원가입 폼 내 복지카드 등록 영역
- **동작**:
  - [복지카드 촬영] 버튼 클릭
  - 카메라로 복지카드 촬영 또는 이미지 업로드
  - AI 서버(`jhserver.shop/api/v1/ocr/welfare-card`)로 이미지 전송
- **결과**: OCR 인식 결과가 회원가입 폼에 자동 입력 (이름, 카드번호 등)

### 3-3. 이메일 인증

- **동작**:
  - 이메일 주소 입력
  - [인증코드 발송] 클릭 → EmailJS로 인증 메일 발송
  - 수신된 인증코드 입력
- **결과**: 이메일 인증 완료

### 3-4. 회원가입 완료

- **동작**: 나머지 필수 정보 입력 후 [가입하기] 클릭
- **결과**: 가입 완료 → 로그인 페이지로 이동

## 시나리오 5: S형 (보호자 공유) 화면 공유

### 5-1. 공유코드 입력

- **URL**: `/S/join`
- **화면**: 보호자 화면 공유 접속 페이지
- **동작**:
  - 메인 랜딩에서 **S형 (공유 모드)** 선택
  - B형 사용자가 발급한 공유코드 입력
  - [참여하기] 클릭
- **결과**: PeerJS WebRTC 연결, B형 사용자의 화면을 실시간으로 공유받음

### 5-2. 화면 공유 확인

- **화면**: 보호자가 B형 사용자의 쇼핑 화면을 실시간으로 관찰
- **동작**: B형 사용자가 쇼핑을 진행하면 보호자 화면에 실시간 반영
- **결과**: 보호자가 시각장애인의 쇼핑을 원격으로 보조 가능
