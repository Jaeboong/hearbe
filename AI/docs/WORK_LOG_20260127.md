## 2) TTS 응답 생성 근거와 흐름

- 질문: "TTS로 읽을 텍스트를 누가 무엇을 근거로 생성하나"
  - 문제: 실제 동작 확인 없이 LLM 응답을 즉시 TTS로 읽어 사용자 혼동
  - 조치: MCP tool 성공/실패에 따른 후속 응답 분기 및 피드백 필요성 정리
  - 결과: tool 실패 알림 모듈 추가 방향 정립

## 3) 클릭 성공 여부와 실패 피드백

- 질문: "장바구니 담기 버튼이 클릭되지 않는다 / 버튼 못 찾나?"
  - 문제: 로그상 click 성공 True인데 실제 페이지 반영/변화 없음
  - 조치: 결과 확인 후 TTS를 내보내는 구조 필요. 실패 시 사용자에게 안내하도록 알림 모듈 추가
  - 결과: ToolFailureNotifier 모듈 추가, action_feedback과 연계해 실패 안내 발화

## 4) 검색 결과 읽기 (n개 더 읽기/중복 읽기)

- 질문: "4번까지만 읽고 더 읽고 싶은지 묻고, n개 더 읽어줘는 동적으로 처리"
  - 문제1: "3개 더 읽어줘" 시 1번부터 다시 읽음
  - 문제2: 추가 읽기 시 리뷰/배송/할인 등 모든 속성까지 읽어버림
  - 조치:
    - 검색 결과 읽기 로직에서 start_index/next_index 유지
    - 기본 읽기 항목을 이름+가격만으로 통일
    - 속성 질의 시에만 해당 속성 읽도록 분리
  - 결과: 기본 읽기 통일, 추가 읽기 중복 문제 해결

## 5) 가격/퍼센트 TTS 숫자 읽기 개선

  - 결과: 가격/퍼센트는 원/퍼센트로 정상 읽기

## 6) HTML 파서 추출 필드 통제

## 7) 검색 인사이트 기반 질의 응답

- 질문: "할인율 가장 높은 상품/가격 가장 낮은 상품/내일 도착 상품 알려줘"
  - 문제: 추출된 속성이 있어도 인사이트 질의가 처리되지 않음
  - 조치: search_insights 모듈로 필터/최솟값/최댓값 계산 로직 추가
  - 결과: 인사이트 질의 응답 가능

## 8) 상품 선택 (n번째 상품 선택 실패)

- 질문: "목록에서 n번째 상품 선택 시 클릭되지 않는다"
  - 문제1: 규칙 선택 로직이 숫자/한글(1~10)만 처리
  - 문제2: '서른번째' 같은 표현이 파싱 실패로 LLM fallback → commands 없음
  - 조치:
    - 한국어 숫자 파서 추가 (삼십/서른/백 등 대응)
    - selection/search_insights/select 규칙에서 공통 파서 사용
  - 결과: 30번째, 100번째 등도 정상 처리 가능

## 9) LLM fallback 처리 개선

- 질문: "rule로 못 찾으면 LLM fallback. LLM도 모르면 추론해 응답"
  - 문제: LLM 응답이 commands 없을 때 무조건 "모르겠다"로 회귀
  - 조치: LLM이 생성한 텍스트가 있으면 그대로 응답하도록 fallback 개선
  - 결과: commands 없는 경우에도 자연스러운 응답 가능

## 10) 스페이스바 barge-in (TTS 중단)

- 질문: "스페이스 누르면 말하던 것 끊고 새 명령을 우선 처리"
  - 문제: 서버/클라이언트 중단 처리 미흡, 재생은 계속됨
  - 조치:

    - 클라이언트: HOTKEY_PRESSED 시 TTS 즉시 stop + interrupt 전송
  - 결과: 스페이스바로 즉시 끊기도록 동작하도록 설계/구현
