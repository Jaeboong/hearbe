## 1) WebSocket MCP Tool Calls 브로드캐스트 버그 수정

### 질문/요청
"배포 서버에서 A컴퓨터, B컴퓨터가 각각 연결했을 때 A컴퓨터의 MCP 명령이 B컴퓨터에도 전달되는 것 같다. 유저별로 분리가 안 되는 것 같은데 맞나?"

### 문제
`api/ws/sender.py`의 `send_tool_calls` 함수에서 tool_calls 메시지를 전송할 때 `broadcast()`를 사용하고 있었음.

```python
# 문제가 된 코드 (sender.py:77-80)
# TODO: [TEMP] Broadcasting to all clients for testing
# In production, send only to the originating session
logger.info(f"[BROADCAST] Sending {len(commands)} tool calls to all clients")
await self._connections.broadcast(msg)
```

- `broadcast()`는 연결된 **모든 클라이언트**에게 메시지를 전송
- 따라서 A 유저가 "장바구니 담기" 명령을 내리면, B 유저의 브라우저에서도 동일한 MCP 명령이 실행됨
- 개발/테스트 용도로 임시 코드가 배포 서버에 그대로 반영된 상태

### 조치

`broadcast()`를 `send_message(session_id, msg)`로 변경하여 해당 세션에만 메시지 전송:

```python
# 수정된 코드
logger.info(f"Sending {len(commands)} tool calls to session {session_id}")
await self._connections.send_message(session_id, msg)
```

### 변경 파일
- `AI/api/ws/sender.py` (line 77-80)

### 결과
- 각 유저(세션)별로 MCP tool_calls가 독립적으로 전송됨
- A 유저의 명령은 A 유저에게만, B 유저의 명령은 B 유저에게만 전달
- 멀티유저 환경에서 정상 동작

---

## 관련 아키텍처 참고

### WebSocket 메시지 전송 방식

| 함수 | 동작 | 용도 |
|------|------|------|
| `send_message(session_id, msg)` | 특정 세션에만 전송 | 일반적인 1:1 통신 |
| `broadcast(msg)` | 모든 연결된 클라이언트에 전송 | 공지, 시스템 메시지 등 |

### 세션 격리가 필요한 메시지 타입
- `TOOL_CALLS`: MCP 명령 (수정 완료)
- `TTS_CHUNK`: 음성 응답
- `ASR_RESULT`: 음성 인식 결과
- `STATUS`: 상태 메시지
- `ERROR`: 에러 메시지

위 타입들은 모두 `send_message()`를 사용하여 세션별로 격리되어 있음.
