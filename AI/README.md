## 1. 음성 인식 (ASR - Automatic Speech Recognition)

### 1.1 실시간 음성-텍스트 변환
- 사용자가 녹음 중인 음성을 청크(chunk) 단위로 실시간 스트리밍하여 서버에서 텍스트로 변환합니다.
- **Faster-Whisper(turbo)** 모델을 사용하여 GPU 가속 기반의 빠른 인식을 제공합니다.
- 한국어 음성을 높은 정확도로 인식하며, 쇼핑 관련 용어(상품명, 브랜드, 가격 등)에 최적화합니다.

### 1.2 음성 명령 인식
- "검색해줘", "장바구니에 담아줘", "결제할게" 등의 쇼핑 명령어를 인식합니다.
- 자연스러운 구어체 표현도 명령으로 해석합니다.

---

## 2. 자연어 처리 (NLU - Natural Language Understanding)

### 2.1 의도 분석
- 사용자 발화에서 쇼핑 의도를 파악합니다.
  - 상품 검색, 상품 비교, 장바구니 추가, 결제 진행 등
- 문맥을 고려하여 불완전한 문장도 이해합니다.
  - 예: "그거 말고 두 번째 거" → 이전 검색 결과의 두 번째 상품 선택

### 2.2 개체명 인식 (NER)
- 발화에서 상품명, 브랜드, 수량, 가격 범위 등의 정보를 추출합니다.
  - 예: "나이키 운동화 10만원 이하로 찾아줘" → 브랜드: 나이키, 카테고리: 운동화, 가격: ~100,000원

---

## 3. 명령 생성 (LLM Planner)

### 3.1 자연어 → 실행 명령 변환 (대화형)
- 사용자의 음성 명령을 브라우저 자동화(MCP) 명령으로 변환합니다.
- 각 단계 실행 후 결과를 사용자에게 안내하고, 다음 행동을 확인받습니다.

**예시 플로우:**
```
사용자: "쿠팡에서 물티슈 찾아줘"

1. 쿠팡 사이트 접속 → "물티슈" 검색
2. 검색 결과 안내: "총 50개 상품이 있습니다. 최저가 3,900원부터 있고,
   브랜드는 크리넥스, 깨끗한나라 등이 있습니다. 어떤 조건으로 찾아드릴까요?"
3. 사용자 응답: "제일 싼 거"
4. 가격 낮은순 정렬 → 첫 번째 상품 상세 안내
5. 사용자 확인 후 장바구니 추가
```

### 3.2 플로우 엔진 위임 판단
- 일반 명령은 즉시 MCP로 실행합니다.
- 회원가입/결제 요청은 Flow Engine으로 위임하여 단계별 진행합니다.

## 4. 음성 합성 (TTS - Text to Speech)

### 4.1 TTS 모델 선정

프로젝트 요구사항 (한국어, 실시간 저지연, 고품질)에 따른 TTS 후보:

| 순위 | 모델 | 지연시간 | 한국어 | 가격 | 비고 |
|------|------|----------|--------|------|------|
| 1 | **ElevenLabs Flash v2.5** | ~75ms | O (32개 언어) | 월 10,000자 무료 | 실시간 에이전트 최적화 |
| 2 | **MiniMax Speech-02-HD** | - | O (악센트 인식) | $0.05/1,000자 | 아시아 언어 특화, 저렴 |
| 3 | **Cartesia Sonic 2** | **40ms** | O | - | 업계 최저 지연 |
| 4 | **CosyVoice 2.0** | 150ms | O | 무료 (오픈소스) | 자체 호스팅 가능 |

**개발/MVP 단계:** MiniMax Speech-02 (저렴, 한국어 품질 우수)
**프로덕션 단계:** ElevenLabs Flash v2.5 (안정성, 문서화, 실시간 성능)
**보안 중시:** CosyVoice 2.0 자체호스팅 (결제 데이터 처리 시)

### 4.2 실시간 음성 스트리밍
- 생성된 음성을 청크 단위로 실시간 스트리밍하여 로컬에 전송합니다.
- 전체 음성 생성 완료 전에 재생을 시작하여 응답 지연을 최소화합니다.

### 4.3 상품 정보 안내 및 질의응답
- OCR 기반 raw 정보와 카테고리별 핵심 정보를 요약하여 안내합니다.

| 카테고리 | 안내 정보 |
|----------|-----------|
| **공통** | 상품명, 가격, 리뷰 평점, 배송 정보 |
| **식품** | 소비기한, 용량, 원산지 |
| **전자제품** | 모델명, 주요 스펙, 보증 기간 |
| **의류** | 사이즈, 색상, 소재 |

- 사용자의 추가 질문에 상세 정보로 응답합니다.
  - 예: "이거 유통기한 언제야?" → "2025년 8월 15일까지입니다."

## 5. 이미지 인식 (OCR - Optical Character Recognition)

### 5.1 상품 이미지 분석
- 상품 이미지에서 색상, 형태, 텍스트 등의 정보를 추출합니다.
- 사용자에게 상품의 시각적 특징을 음성으로 설명합니다.
  - 예: "파란색 반팔 티셔츠이고, 가슴에 흰색 로고가 있습니다."

### 5.2 인증 이미지 처리 (OCR)
- CAPTCHA, 보안문자, 결제 키패드 등의 인증 이미지를 인식합니다.
- 키패드 영역 검출 후 숫자 매핑 및 정렬을 수행합니다.

---

## 6. 쇼핑 플로우 엔진 (Flow Engine)

### 6.1 사이트별 자동화 시나리오
- 주요 쇼핑몰(쿠팡, 네이버쇼핑, 11번가 등)별 플로우를 정의합니다.
- 각 단계(step)는 다음을 포함합니다:

```python
{
    "flow_type": "signup | checkout",
    "site": "coupang | naver | 11st",
    "steps": [
        {
            "prompt": "TTS 안내 문구",
            "required_fields": ["name", "phone"],
            "action": "MCP 호출 액션",
            "validation": "성공/실패 판단",
            "fallback": "실패 시 다음 안내"
        }
    ]
}
```

### 6.2 결제 안전 확인 (Policy/Guard)
- 결제/회원가입 전 확인 단계를 필수로 삽입합니다.
- 상품명, 가격, 수량, 배송지를 음성으로 최종 확인합니다.
- 민감 정보(비밀번호 등)는 로컬에서만 입력하도록 강제합니다.

---

## 7. 세션 및 컨텍스트 관리

### 7.1 대화 맥락 유지
- 쇼핑 세션 동안 현재 사이트, 진행 중 플로우, 검색 기록 등을 저장합니다.
- "아까 그 상품", "방금 본 거" 등의 대명사 참조를 해석합니다.

### 7.2 즐겨찾기 및 선호도 관리 (옵션)
- 자주 구매하는 상품, 선호 브랜드, 단골 쇼핑몰을 즐겨찾기로 저장합니다.
- "평소 먹던 우유 주문해줘" 같은 요청에 즐겨찾기 기반으로 빠르게 응답합니다.
- 검색 시 선호 브랜드를 우선 추천합니다.

## 8. 실시간 통신

### 8.1 WebSocket (스트리밍 데이터)
- 대용량/실시간 데이터의 양방향 스트리밍에 사용합니다.

| 방향 | 메시지 타입 | 설명 |
|------|-------------|------|
| 로컬 → 서버 | `audio_chunk` | 녹음 중인 음성 청크 실시간 전송 |
| 서버 → 로컬 | `stt_result` | STT 변환 텍스트 결과 |
| 서버 → 로컬 | `tool_calls` | LLM 생성 MCP 명령 |
| 서버 → 로컬 | `flow_step` | 플로우 엔진 단계 정보 |
| 서버 → 로컬 | `tts_chunk` | TTS 음성 청크 실시간 전송 |
| 양방향 | `status` | 상태 정보 |

### 8.2 HTTP API
- 인증/토큰 갱신/헬스체크에 사용합니다.

| 엔드포인트 | 설명 |
|------------|------|
| `POST /auth/login` | 로그인 시작 |
| `GET /auth/callback` | 로컬 콜백용 (OAuth) |
| `POST /auth/token` | code → access/refresh 토큰 |
| `POST /auth/refresh` | 토큰 갱신 |

## 10. 전체 동작 플로우

```
1. 데스크탑 앱 실행 → 로컬 콜백 서버 열기 (http://127.0.0.1:port/callback)
2. 앱이 자사 로그인 URL을 브라우저로 오픈
3. 사용자 로그인 완료 → 서버가 리다이렉트로 앱 로컬 콜백에 auth_code 전달
4. 앱이 auth_code를 서버로 전송(HTTP) → access/refresh 토큰 발급
5. 앱이 WebSocket 연결 → 세션 생성
6. V 키 → 오디오 스트림 전송(WS) → STT → 텍스트 반환
7. 서버 LLM(GPT-5-mini)로 명령 생성
8. "일반 명령"이면 바로 MCP 실행 결과 수신
9. "회원가입/결제 플로우"면 사이트별 플로우 엔진이 단계별로 진행
10. 각 단계마다 TTS 안내(빈 칸/필수정보/확인)
11. 결제 인증: 사용자 유형별 분기
    - 시각장애인 카드: 음성 기반/대체 인증
    - 일반 사용자: 직접 2차 비밀번호 입력(로컬)
12. 완료/실패 결과를 TTS로 안내
```

---

## 11. 보안 및 프라이버시

- 음성 데이터 저장 없음 (스트리밍 처리 후 폐기)
- 결제/2차 비밀번호는 로컬 입력만 허용
- 토큰은 단기 access + refresh 로테이션
- TTS 서비스 자체호스팅 옵션 제공 (CosyVoice 2.0)
