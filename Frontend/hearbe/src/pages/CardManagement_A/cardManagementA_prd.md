# CardManagementA PRD

## 목적
복지카드 정보를 등록/수정하고 저장된 정보를 조회한다.

## 범위
- A 타입 복지카드 변경 화면(CardManagementA)
- 복지카드 정보 입력 및 저장 API 연동
- 저장 후 최신 정보 조회

## 사용자 흐름
1. 복지카드 변경 페이지 진입
2. 현재 복지카드 정보 확인
3. 변경하기 클릭
4. 카드 촬영 후 정보 입력/수정
5. 저장 버튼 클릭
6. 서버 저장 성공 시 최신 정보 표시

## API 명세
- 저장/수정
  - Path: /welfare
  - Method: PUT
  - Request Body:
    {
      "card_company": "신한카드",
      "card_number": "1234-5678-9012-3456",
      "issue_date": "2023-01-01",
      "expiration_date": "2028-01-01",
      "cvc": "123"
    }
  - Response:
    {
      "code": 200,
      "message": "복지카드가 등록/수정되었습니다.",
      "data": { /* 동일한 형태 */ }
    }
- 조회
  - Path: /welfare
  - Method: GET
  - Response: 저장된 복지카드 정보

## 입력/검증
- 모든 필드 필수
- 날짜 형식: YYYY-MM-DD
- 카드번호 형식: ####-####-####-####
- CVC: 3자리 숫자

## 상태 및 메시지
- 저장 실패 시 오류 메시지 표시
- 저장 성공 시 최신 정보로 갱신

## 범위 외
- OCR/자동 인식 정확도 개선
- 카드번호 마스킹 정책 변경
