# FindIdA PRD

## 목적
복지카드 정보로 아이디를 조회하는 아이디 찾기 기능을 제공한다.

## 범위
- A 타입 아이디 찾기 화면(FindIdA)
- 복지카드 정보 입력 및 아이디 조회 API 연동
- 조회 성공/실패 결과 표시

## 사용자 흐름
1. 아이디 찾기 화면 진입
2. 복지카드 촬영 버튼 클릭
3. 복지카드 정보 입력 화면 표시
4. 확인 버튼 클릭
5. 서버 조회 결과에 따라 아이디 표시 또는 실패 메시지 표시

## API 명세
- Path: /auth/findId
- Method: POST
- Request Body:
  {
    "welfare_card": {
      "card_company": "신한카드",
      "card_number": "1234567890123456",
      "issue_date": "2023-01-01",
      "expiration_date": "2028-01-01",
      "cvc": "123"
    }
  }
- Response:
  {
    "code": 200,
    "message": "성공적으로 처리되었습니다.",
    "data": {
      "message": "Found ID",
      "username": "blind_test_001"
    }
  }

## 입력/검증
- 모든 복지카드 필드 필수
- 날짜 형식: YYYY-MM-DD
- 카드번호 형식: ####-####-####-####

## 상태 및 메시지
- 조회 중: "조회 중..."
- 성공: "카드 정보가 확인되었습니다" + 아이디 표시
- 실패: "카드 정보가 없습니다" + 재촬영 유도

## 오류 처리
- 네트워크/서버 오류 시 실패 메시지 표시
- 카메라 접근 실패 시 권한 안내 메시지 표시

## 저장/보안
- 아이디 조회 결과는 화면 표시 후 즉시 종료
- 복지카드 정보는 클라이언트에 저장하지 않음

## 범위 외
- OCR/카드 자동 인식 기능
- 복지카드 데이터 암호화 저장
