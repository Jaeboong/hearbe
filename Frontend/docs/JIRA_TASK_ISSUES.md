## 📌 컨벤션 요약

| 구분 | 형식 |
|------|------|
| **에픽** | {도메인명} |
| **스토리** | `Feat: {기능명}` + 설명: [Role] 사용자는 {목적}을 하기 위해 {기능명}을 사용하고 싶다 |
| **태스크** | 회의, 미팅 등 코드 외 업무 |
| **버그** | QA/운영 중 발견한 오류 |

### Issue Title Type
- `Meeting`: 회의
- `Docs`: 문서 작성
- `Feat`: 기능 구현
- `Fix`: 오류/버그 수정
- `Refactor`: 코드 리팩토링
- `Style`: UI 작업
- `Rename`: 파일명/폴더명 수정
- `Remove`: 파일/기능 삭제

### Story Point 기준
| SP | 설명 |
|----|------|
| 1 | 아주 단순, 눈으로 봐도 해결 |
| 3 | 기준 스토리, 평균 난이도 |
| 5 | 복잡한 로직, 통합 필요 |
| 8 | 매우 복잡, 협업 필요 |
